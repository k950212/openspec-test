## Why

目前商品列表已經有搜尋、分類與排序，但使用者還不能依價格快速縮小範圍，找特定預算內商品時效率不佳。新增價格區間篩選可以讓瀏覽體驗更接近實際電商場景，也能提升現有 36 筆商品資料的可探索性。

## What Changes

- 在商品列表頁新增價格區間篩選，讓使用者可以依最低價與最高價限制商品結果。
- 價格區間篩選會和現有搜尋、分類與排序一起運作，並即時更新商品結果。
- 當篩選後沒有符合商品時，頁面要顯示清楚的 empty state 與重設篩選入口。
- 提供清除價格區間的操作，讓使用者可以快速回到完整商品列表。

## Capabilities

### New Capabilities
- `price-range-filter`: 定義商品列表的最低價與最高價篩選行為、驗證與清除流程。
- `product-catalog`: 定義商品列表在套用價格區間後的結果呈現與 empty state 行為。

## Impact

- 影響前端商品列表頁的篩選狀態與畫面呈現。
- 預計修改 `src/views/HomeView.vue` 與相關商品列表計算邏輯。
- 不需新增後端 API，使用既有前端商品資料即可完成。
